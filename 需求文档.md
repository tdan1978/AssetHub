IT 资产管理系统 (ITAM) 综合需求规格说明书

1. 项目概述

1.1 项目背景

随着公司规模扩大，IT 资产（服务器、笔记本、软件授权、网络设备）数量激增。传统 Excel 记录方式存在信息滞后、权责不明、资产流失等痛点。

1.2 项目目标

构建一套基于 Python + Vue + MySQL 的全生命周期管理系统，实现资产“入、出、移、修、盘、退”的闭环管理，确保账实相符。

2. 技术架构方案

后端 (Backend): Python 3.9+ (推荐 FastAPI 或 Django REST Framework)

前端 (Frontend): Vue.js 3 + shadcn-vue (PC 端) + 移动端适配页面 (用于扫码)

数据库 (Database): MySQL 8.0 (InnoDB)

中间件: Redis (用于登录 Token 缓存及操作频率限制)

核心工具: Pandas (Excel 处理)、QRCode (二维码生成)、Bcrypt/Argon2 (加密)

4. 资产全生命周期管理 (核心逻辑)

4.1 资产状态流转图

系统严格限制状态流转路径：

闲置 → (领用申请) → 在用

在用 → (退库) → 闲置 或 维修中

在用/闲置 → (维修申请) → 维修中 → (修复) → 闲置

任何状态 → (报废申请) → 待报废 → (审批通过) → 已报废

4.2 功能模块细化

4.2.1 资产入库与档案

单一录入: 必填项：SN (全局唯一)、资产名称、分类、采购日期。

自动编号: 系统按规则 [类型前缀]-[年月日]-[4位流水] 自动生成内部管理号。

批量导入: * 两阶段校验: 1. 静态校验 (格式/必填项); 2. 业务校验 (SN 重复检查/分类匹配)。

错误回显: 导入失败需下载带有“失败原因”列的原始 Excel。

附件关联: 价值 > 5000 元的资产强制要求上传发票/合同照片。

4.2.2 领用、调拨与退库

领用: 仅 闲置 资产可见。需记录领用人、部门、预计归还时间、存放位置。

调拨: 跨部门转移，需原部门管理员与新部门管理员双重确认。

退库: 必须填写“设备状况”。若勾选“损坏”，系统自动在维修模块生成一条“待处理”记录。

4.2.3 软件资产管理 (SAM)

授权池: 记录 Key、总授权数、已分配数、有效期。

库存预警: 剩余数 < 10% 或 距离到期 < 30 天时，仪表盘标红预警。

绑定逻辑: 记录软件 License 绑定在哪个具体的硬件 SN 上。

4.2.4 维保与报修

维保管理: 记录供应商信息、维保合同、厂家报修电话。

维修记录: 累加维修费用至资产详情，用于计算该资产的“持有总成本”。

4.2.5 智能盘点 (Stocktake)

任务下发: 管理员按部门、区域或分类发起盘点。

移动端扫码: 管理员通过手机扫描资产二维码：

匹配: 标记“盘盈”或“正常”。

缺失: 任务结束未扫描的自动标记为“盘亏”，需提交原因说明。

5. 业务规则与逻辑约束 (Critical)

SN 唯一性: 数据库层面 sn 字段设置 UNIQUE 索引，禁止任何重复记录。

软删除机制: 所有删除操作（用户、资产、分类）均为 is_deleted = 1，不可物理删除，确保历史日志完整性。

报废限制: 严禁将“在用”资产直接改为“已报废”，必须经过“退库”或“申请审批”流程。

操作审计: 任何资产字段的修改必须在 asset_logs 中记录：操作人 | 字段名 | 旧值 | 新值 | 时间。

6. UI/UX 交互规格

6.1 PC 端 (管理后台)

仪表盘: * 四个核心指标：总数、在用率、待维修数、本月报废数。

饼图：资产按分类占比。

列表：最近 7 天即将过期的维保提醒。

列表交互: 支持表头自定义显示隐藏列；支持一键导出当前筛选结果为 Excel。

6.2 移动端 (扫码工具)

扫码后展示:

员工扫码: 仅展示资产名称、规格、维保期，提供“报修申请”和“交还申请”按钮。

管理员扫码: 展示完整信息，提供“状态变更”、“位置移动”按钮。

7. 非功能性需求

7.1 安全性 (Security)

加密: 密码采用 Bcrypt 加密，Token 采用 JWT，设置 24 小时过期。

限流: 对登录和批量导出接口设置频率限制，防止撞库或恶意下载。

7.2 性能 (Performance)

查询: 单表 5 万行数据下，列表翻页响应需 < 500ms。

并发: 领用操作需使用数据库行锁，防止同一资产同时被多人领取。

7.3 可靠性 (Reliability)

备份: 数据库每日凌晨自动执行 mysqldump 并异地存储。

日志: 后端记录所有 4xx/5xx 错误日志以便快速排查
